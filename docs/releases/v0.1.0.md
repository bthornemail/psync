# FTF Reference Implementation v0.1.0

Initial public release of the **Frozen Trace Fabric (FTF)** reference implementation.

## Highlights

### Frozen Wire Protocol

The protocol is specified in:

```text
PROTOCOL.md
```

Key invariants:

- canonical CBOR projection
- multihash SHA-256 addressing
- Ed25519 signatures over message hashes
- deterministic replay ordering `(t, mh)`
- append-only message streams

## Full Body Support

The reference implementation supports all protocol body kinds:

```text
put
use
xform
attest
revoke
alias_claim
```

Both the Haskell CLI and the Rust verifier implement these bodies.

## Cross-Language Verification

The repository includes an independent Rust verifier:

```text
rust/ftf-verify
```

The conformance harness verifies:

- canonical hashing
- signature validation
- deterministic replay
- alias resolution
- body-kind compatibility

Harness command:

```bash
bash verify-wire.sh
```

Expected result includes:

```text
verified=5
rejected=0
[verify-wire] ok wire conformance
```

## Static Observatory

A static inspection UI is included under:

```text
public/
```

Features:

- deterministic trace visualization
- artifact and event tables
- replay frontier scrubber
- alias chain inspection
- proof receipt viewer
- NDJSON trace ingestion

The UI supports embedded data, hosted assets, and drag-and-drop inspection.

## Proof Surface

Protocol semantics are documented via bounded receipts in:


Included receipts:

- wire conformance

- attest semantics
- Rust full-body conformance
