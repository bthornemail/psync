## Goal

`v0.2.0` introduces **closed mnemonic spaces** as a workflow layer on top of
the frozen `v0.1.x` protocol kernel.

The kernel remains unchanged:

- `put`
- `use`
- `xform`
- `attest`
- `revoke`
- `alias_claim`
- `trace` (query/projection)

Spaces add ergonomics and identity workflows without mutating protocol wire
semantics.

## Product Model

A space is local-first and minimally stateful:

- space identity
- space config
- member identities
- topic logs and CAS blobs

Suggested on-disk layout:

```text
.ftf/
  config.json
  spaces/
    <space>/
      space.json
      members.json
      aliases.ndjson
      topics/
      cas/
      keys/
```

## `ftf space`

### `ftf space new <name>`

Creates a new local space.

Options:

- `--mnemonic` print generated mnemonic
- `--no-mnemonic` generate key file only
- `--realm <realm>` explicit realm override
- `--path <dir>` custom space root path

Example:

```bash
ftf space new lab --mnemonic
```

Example output:

```text
Space: lab
Realm: ftf:lab
Mnemonic: orchard lava arrow ...
Space ID: 1220...
Created: .ftf/spaces/lab
```

### `ftf space show <name>`

Displays:

- realm
- space id
- member count
- topic count
- CAS object count
- active identity

### `ftf space ls`

Example:

```text
lab     realm=ftf:lab     members=3
alpha   realm=ftf:alpha   members=1
```

### `ftf space export <name>`

Modes:

- `--public`
- `--bundle`
- `--member <id>`

### `ftf space import <file>`

Imports:

- public metadata
- invitation bundle
- backup bundle

### `ftf space member add <space> <name> --pubkey <hex>`

Optional:

- `--role owner|operator|auditor|writer|reader|relay|bot`

### `ftf space invite <space> --role <role>`

Creates a signed invitation bundle (stdout or file redirect).

## `ftf identity`

### `ftf identity new`

Generates standalone identity:

- mnemonic
- public key
- peer id

### `ftf identity derive --space <space> --path <path>`

HD-style derivation.

### `ftf identity derive --space <space> --role <role>`

Role-based deterministic derivation path alias.

### `ftf identity show --space <space>`

Shows active local signer for the space.

### `ftf identity use --space <space> --member <id>`

Sets active signer.

## Provenance Wrappers

### `ftf put <path> --space <space>`

Behavior:

- stores blob in CAS
- emits signed `put`
- appends to default or selected topic

Options:

- `--topic <topic>`
- `--name <label>`
- `--tag <tag>`

### `ftf use --space <space> --input <hash-or-path> --purpose <text>`

Friendly wrapper over `use` body creation/signing.

### `ftf xform <label> --space <space> --input <h|path> --output <h|path>`

Behavior:

- resolves paths to hashes
- stores outputs when path input is a local file
- emits signed `xform`

Options:

- `--tool <hash>`
- `--params <file>`
- `--receipt <file>`
- `--topic <topic>`

### `ftf attest <hash-or-path-or-alias> --claim <claim> --space <space>`

Options:

- `--evidence <hash>` (repeatable)
- `--evidence-file <path>` (repeatable)
- `--topic <topic>`

### `ftf revoke <hash-or-path-or-alias> --reason <reason> --space <space>`

Options:

- `--superseded-by <hash>`
- `--topic <topic>`

Semantics remain annotation-only (non-destructive).

### `ftf alias set <logical-id> <hash> --space <space>`

### `ftf alias get <logical-id> --space <space>`

### `ftf alias ls --space <space>`

## Query and Verification

### `ftf trace <hash-or-alias-or-path> --space <space>`

Resolution order:

1. path -> CAS hash
2. alias -> target hash
3. raw hash

Output modes:

- NDJSON default
- `--pretty` terminal tree

### `ftf verify --space <space>`

Checks:

- signatures
- `prev` linkage
- monotone `t`
- alias chain validity

Options:

- `--topic <topic>`
- `--strict`
- `--json`
- `--check-cas` (optional materialization checks)

## Sync (Bundle-First)

`v0.2.0` keeps sync simple and local-first:

- `ftf sync pull <space> <bundle-or-peer>`
- `ftf sync push <space> <bundle-or-peer>`
- `ftf sync heads <space>`

Initial transport target is file/bundle exchange (no daemon requirement).


## Implementation Constraints

- Keep protocol kernel unchanged.
- Keep workflow local-first and offline-capable.
- Prefer deterministic files (`ndjson`, `json`) over opaque mutable stores.
- Avoid consensus/token/blockchain semantics in spaces layer.
